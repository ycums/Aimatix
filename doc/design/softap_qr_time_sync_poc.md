# AIM-36 SoftAP + QR 二段階方式 時刻反映 SPIKE 設計方針書

## 目的 / スコープ
- iPhone等のスマートフォンから、アプリ配布なしで M5Stack へ「ローカル時刻を一方向反映」する実用手段の検証（PoC）。
- 本資料は SPIKE（PoC）段階の設計方針。純粋ロジック層は汚染せず、`src/` 層内のアダプタ実装に限定。

## 前提・制約（PoC）
- SoftAP は 60秒の公開時間窓、先着1クライアント許可。成功または時間窓切れで停止。
- トークン・PSK はランダム生成（PoCでは64bit相当）。
- ドリフト閾値チェックは PoC では無制限（全許容）。
- HTTPサーバは Arduino `WebServer` を使用（最速優先）。

## ユーザーフロー（二段QR）
1. 装置側で SoftAP を起動（SSID/PSK をランダム生成）。
2. 画面に Wi‑Fi接続用QR を表示（`WIFI:T:WPA;S:<SSID>;P:<PSK>;H:false;;`）。
3. スマホが AP に接続すると、自動で URL QR に切替（`http://192.168.4.1/sync?t=<token>`）。
4. `/sync` HTML が `Date.now()` と `getTimezoneOffset()` を取得し `POST /time/set` を自動送信。
5. 受理・適用成功で AP を停止し、実装では即時に `MAIN_DISPLAY` へ復帰（成功画面の固定表示は行わない）。

## QR 仕様
- Wi‑Fi QR: `WIFI:T:WPA;S:<SSID>;P:<PSK>;H:false;;`
  - エスケープ: SSID/PSK に `\\` `;` `,` `:` を含む場合はバックスラッシュでエスケープ。
- URL QR: `http://192.168.4.1/sync?t=<token>`
- 表示: `ricmoo/QRCode` で QR を生成し、M5GFX でドット描画。画面サイズに応じ自動スケール。

### PoC 表示設定（最終）
- ECC: LOW（誤り訂正7%）
- quiet zone: 2 モジュール（大きさ優先）
- 画面余白: 上端 10px を見出し用に確保
- 配色: 黒背景 + 橙ドット（可読性よりサイズ優先の選定）
- バージョン目安: Wi‑Fi=7、URL=5（本実装でも同目安を採用）
- 切替: APクライアント接続イベントで Wi‑Fi → URL に自動切替

## API 仕様
- `GET /sync`
  - 入力: `?t=<token>`（ワンタイム）
  - 出力: HTML（JSが `POST /time/set` を自動実行）
- `POST /time/set`
  - 入力(JSON): `{ "epochMs": <int>, "tzOffsetMin": <int>, "token": "<hex>" }`
  - 受理条件（PoC）:
    - `token` 一致
    - AP時間窓内（<= 60秒）
    - レート制限（1回/起動）
    - ドリフト閾値: 無制限（PoC）
  - 入力バリデーション（本実装方針）:
    - `epochMs` 下限: `2025-01-01 00:00:00 UTC`（`1735689600000`）以上
    - `epochMs` 上限: `sizeof(time_t)==4` の場合 `2038-01-19 03:14:07 UTC`（`2147483647000`）以下、`sizeof(time_t)==8` の場合 `2100-01-01 00:00:00 UTC`（`4102444800000`）以下
    - `tzOffsetMin` 範囲: `[-14*60, +14*60]` 分（`[-840, +840]`）
  - 出力:
    - 200: `Time applied`
    - 400: `reason: "time_out_of_range" | "tz_offset_out_of_range" | "invalid_token"`
    - 4xx/5xx: 理由文字列

## セキュリティ要件（PoC → 本実装での強化方針）
- PoC:
  - トークン: 64bit相当（16桁Hex）
  - PSK: 64bit相当（16桁Hex）
  - 最大接続: 1、露出: 60秒、レート制限: 1回/窓
- 本実装の推奨:
  - トークン: 128bit以上
  - PSK: 128bit相当以上、記号を含む高強度
  - 接続性警告の緩和（下記「既知事象」参照）
  - 監査ログ/失敗回数に応じたバックオフ

## 既知事象 / 運用上の注意
- スマホ側に「ネットワークに接続できませんでした」等の警告が出る場合がある。
  - 理由: SoftAP はインターネットに出られないため、OSの接続性チェックで失敗と判定されやすい。
  - 回避: 一時的にセルラー無効化 or Wi‑Fiアシスト無効化、または `http://192.168.4.1/sync?t=...` を直接開く。
  - 改善（任意）: `/generate_204`（204応答）や `hotspot-detect` 互換パスを用意し、接続性チェックを満たして警告を抑止。

- 環境: `env:m5stack-core2-aim36-spike`（歴史的参照。現行は本番環境で `src/SoftApTimeSyncController.cpp` を使用）
- エントリ: `src/spikes/aim36/main.cpp`（歴史的参照）
- 現行実装: `src/SoftApTimeSyncController.cpp`
- 画面: Amber CRTトーン、Wi‑Fi QR → URL QR 切替は APクライアント接続イベントで自動。
- `/time/set` 成功後は `Local` と `TZ` の情報を Serial ログに出力し AP 停止（本実装では画面固定表示は行わず即時復帰）。

## UI文言（本実装差分）
- タイトル区切り記号: `|` → `>` に変更
- Step1 タイトル: "TIME SYNC > JOIN AP"
- Step2 タイトル: "TIME SYNC > OPEN URL"
- 成功: "TIME SYNC > DONE"（固定画面は出さずログのみ）
- エラー: "TIME SYNC > ERROR"
- ボタンヒント: A="REISSUE" / B=未使用 / C="EXIT"

## 運用手順（運用者向け）

本セクションは現場運用における手順の定義です。UI仕様の詳細は `doc/spec/time_sync_ui_spec.md` を参照してください。

### 前提条件
- 装置は時刻同期画面（TIME SYNC）に入れる状態であること
- 近接環境（スマホと装置の距離2m以内推奨）
- 同期中の公開時間窓は 60 秒、先着 1 クライアントのみ

### 手順
1. 装置で時刻同期を開始する（設定メニュー等から TIME SYNC を選択）。
2. 画面に表示された Wi‑Fi 接続用QR（QR1）をスマホで読み取り、SSID/PSK で SoftAP に接続する。
   - 接続性の警告が出る場合があるが想定内（後述）。
3. スマホが接続されると装置側は自動で URL 用QR（QR2）に切り替わる。
4. QR2 の URL（`http://192.168.4.1/sync?t=<token>`）を開くと、ページ読み込みにより自動で `POST /time/set` が送信される。
5. 装置側で時刻が適用されると SoftAP は停止し、即時に `MAIN_DISPLAY` へ復帰する（成功固定画面は表示しない）。
6. 失敗時は SoftAP を停止しエラーメッセージを 2 秒表示後に `SETTINGS_MENU` へ戻る。

### ボタン操作（装置側）
- A: `REISSUE`（トークン/PSKを再発行し SoftAP を再開始）
- C: `EXIT`（SoftAP を停止し `SETTINGS_MENU` へ戻る）

### OS別の注意（スマホ側）
- iOS: セルラーを一時的に無効化、または Wi‑Fiアシストを無効化すると接続性警告を抑制しやすい。
- Android: 接続性チェックによりポータル誘導や「インターネットなし」警告が出ることがある。URL を手動で開く（`http://192.168.4.1/sync?t=<token>`）。

### 成功確認と再試行
- 成功時: 装置が `MAIN_DISPLAY` へ復帰し、Serial ログに Local/TZ が出力される。
- 失敗/タイムアウト時: 画面のエラーメッセージを確認後、A で再発行または C で終了→やり直し。
- 注意: 公開時間窓（60 秒）と 1 回/起動のレート制限（PoC）がある。

## トラブルシュート

代表的な失敗パターンを下表に整理する。

| 症状/表示 | 想定原因 | 対処 |
|---|---|---|
| AP 接続不可（認証失敗/接続拒否） | PSK誤り、距離、最大1接続に到達 | 近接して再試行、Aで再発行、既存接続を切断 |
| URL が開かない/自動遷移しない | OSの接続性チェックでブロック、ポータル誤誘導 | `http://192.168.4.1/sync?t=...` を手動で開く。セルラー/アシストを一時無効化 |
| `AP WINDOW EXPIRED` | 60 秒の公開窓を超過 | Aで再発行して最初からやり直す |
| `TOKEN MISMATCH` | 旧QR（QR2）を参照、トークン不一致 | Aで再発行→最新QR2を再スキャン |
| `TIME OUT OF RANGE` | 端末時計が極端にずれている | 端末の日時を正し、再実行 |
| `TZ OFFSET OUT OF RANGE` | 端末のタイムゾーン情報が異常 | 端末のタイムゾーン設定を見直し、再実行 |
| `APPLY FAILED` | 端末側適用に失敗、内部例外 | 装置のSerialログを確認し再試行。必要に応じ再起動 |
| スマホに「インターネットなし」警告 | SoftAPは外部に出られない設計 | 警告は無視可。必要ならセルラー/アシストを一時無効化 |
| 二段QRが切り替わらない | クライアント接続イベント未検出 | AP接続状態を確認。電波環境を改善し再試行 |

## QRエスケープ/エンコード表（Wi‑Fi/URL/非ASCII）

### Wi‑Fi QR（WIFI 形式）
フォーマット: `WIFI:T:WPA;S:<SSID>;P:<PSK>;H:false;;`

- エスケープ対象と規則（`S:`/`P:`の値内）

| 文字 | 入力例 | エスケープ後 | 備考 |
|---|---|---|---|
| バックスラッシュ `\` | `net\work` | `net\\work` | まず `\` を二重化 |
| セミコロン `;` | `my;ssid` | `my\;ssid` | フィールド区切りのため必須 |
| カンマ `,` | `my,ssid` | `my\,ssid` | 実装互換のため推奨 |
| コロン `:` | `my:ssid` | `my\:ssid` | 実装互換のため推奨 |
| 空白 | `my ssid` | `my ssid` | そのまま可（先頭/末尾空白は非推奨） |
| 二重引用符 `"` | `my"ssid` | `my\"ssid` | 一部リーダー対策で推奨 |

- 非ASCII（日本語・絵文字など）
  - SSID: UTF‑8表現がQR/スマホ側で読み取り可能な場合が多いが、機種差あり。実運用では英数字と一部記号を推奨。
  - PSK: WPA2‑PSKは「8–63 ASCII」推奨。非ASCIIは互換性低下のため非推奨。64桁Hex（256bit）を用いる場合は ASCII のみ。

- エスケープ例

```
S:My\;Cafe\:WiFi,P:pa\\ss\,word
→ WIFI:T:WPA;S:My\;Cafe\:WiFi;P:pa\\ss\,word;H:false;;
```

### URL QR（`http://192.168.4.1/sync?t=<token>`）
- 現行実装: `token` は16桁Hex想定で URLエンコード不要。
- 将来拡張方針: 非ASCIIや空白、予約文字を含むクエリ値は UTF‑8 を前提に Percent‑Encoding（RFC 3986）を適用。

- エンコード対象例（将来拡張時）

| 種別 | 例 | エンコード | 備考 |
|---|---|---|---|
| 空白 | ` "a b" ` | `%20` | プラス `+` ではなく `%20` を採用 |
| 非ASCII | `日本` | `%E6%97%A5%E6%9C%AC` | UTF‑8 → %HH 連結 |
| 予約文字 | `# & = ?` | `%23 %26 %3D %3F` | 一般的なクエリ値の保護 |

- 例（将来、`label` パラメータ追加時）

```
http://192.168.4.1/sync?t=0123abcdEF45ff09&label=%E6%97%A5%E6%9C%AC%20AP
```
